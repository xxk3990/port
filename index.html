Party List using Google Firebase

This is a website I made in November 2016 that allows the user to enter their information if they want to attend a party. Their data is then stored in a database, the one I used was Google Firebase. The user can add their info to the database and then display the list (their info along with the list of people already in the database). The user can also change their status using the checkbox. The default is "Invited", but if the user wants to attend they click the checkbox and it says "Change (Name + School) from invited to attending". I used JavaScript to save the info in the database, display it on the screen, and update the status. I used HTML to make the input boxes, the buttons, and the table containing the database list.

Genetic Algorithm of Hue Grids

This is a project I made in April 2018 in which I wrote code that applied different processes in Genetics to a set of 9 hue grids. I did mutation, crossover, selection, and breeding (either the first grid or 9 new ones). The hue grids themselves have different designs that I chose to implement. I have a range of red, a range of blue, a range of green, and a rainbow color. Each color has a random layout, a vertical stripe layout, a horizontal stripe layout, and a gradient. The muation takes the first grid and mutates it, the crossover takes the first grid and crosses it with another random grid, the select1 takes the first grid and does selection on it, the breed1 breeds a new grid in the first spot, and the breed gen breeds a whole new generation of huegrids that are "children" of the 1st grid, inheriting "chromosomes" from it. You can also adjust the rate at which each square in the hue grid changes and the transparency. This was all done using a JavaScript library called P5.

20 Questions C# Console App

This is a C# console app I made in November 2018 that allows the user to play 20 questions. The project uses File.IO to store names/clues in an external file. Before the game starts, there are three things in the file. The first is a clue and the other two are guesses. The project uses a Binary Search Tree that has each clue as a branch node and the leaf nodes are names of people. The tree also allows the computer to go through and make decisions based on user input and where it is in the tree. The game begins with the computer asking if it is living. The user can then enter either yes or no, and the computer will make the guess based on user response. If the answer to the guess is no, the computer will ask who the user was thinking of, and then ask them what the difference between the two are. Once you enter that, the computer uses recursion to add the new names and clues to their correct place on the tree. In the next game, it will ask you the difference you inputted and then make a guess based on that (depending on what you say, it could also guess the new person you entered). Once you quit the project, those new names and differences are saved to the file.
